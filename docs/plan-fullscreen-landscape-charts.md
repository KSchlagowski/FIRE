# Fullscreen landscape charts (chart zoom overlay)

## Context

Every chart in the app is an inline SVG generated by `chartSVG` / `stackedBarSVG`
(`js/ui.js:125` / `js/ui.js:172`) with a fixed `viewBox` of 440×170 and rendered
inside a ≤480px-wide card. On a phone in portrait that is small: axis labels are
~9 real px and multi-decade series get decimated to ≤120 points.

This feature adds a small **expand button next to every chart**. Tapping it opens
the chart in a **fullscreen overlay**; the user rotates the phone 90° and reads the
chart large and horizontal. Crucially it must also work for users with the OS
auto-rotate lock **on** (very common): in that case the overlay draws the chart
**pre-rotated 90° via CSS**, so physically turning the device is enough — no OS
rotation required.

Nothing is persisted; no engine math changes. This is a pure presentation feature:
one new pure module, UI glue in `ui.js`, CSS, one overlay div in `index.html`.

## Locked decisions

- **D1 — Re-render, don't scale.** The fullscreen view re-renders the chart with a
  viewBox matched to the viewport aspect ratio and a `detail` flag (more grid
  lines, more axis labels, decimation raised to 240 points). Simply CSS-scaling
  the existing 440×170 SVG would scale the 10px labels into billboard text and
  waste the extra pixels on nothing.
- **D2 — Chart builders move to a new pure module `js/charts.js`.**
  `chartSVG`, `stackedBarSVG` and their private helper `formatShort` leave `ui.js`
  (where they are pure already — `formatShort` has no other callers, verified) and
  become a zero-import leaf module, mirroring `analysis.js`' local-`esc()` pattern.
  This is what makes the new sizing/detail options testable in the Node runner
  (`ui.js` itself can never be imported there). `ui.js` imports them back; no other
  module references them today (analysis/simulation receive finished strings).
- **D3 — A registry of rebuild closures in `ui.js`, keyed per chart site.** Charts
  are passed into the pure builders as `chartHTML` strings, so the pure layers
  never change signature. At each call site `ui.js` wraps the chart through a
  `zoomable(key, title, build, {legendHTML})` helper that (a) stores the `build`
  closure in a module-scope `Map` and (b) returns the chart wrapped with the
  expand button. The overlay later calls `build({width, height, detail, maxPoints})`
  to re-render at any size. Keys are stable per site (`'dash-portfel'`,
  `'sym-nadplata'`, …) so the dynamic result swaps (sliders, inputs) **overwrite**
  their entry instead of leaking; the registry is cleared in `route()`.
- **D4 — One delegated click listener, not per-render wiring.** Several charts live
  inside dynamically swapped subtrees (`#dwz-result`, `#sym-overpay-result`,
  `#sym-loan-result`-style swaps) that get `innerHTML`-replaced without re-running
  the renderer's listener block. A single `document`-level delegated listener for
  `[data-chart-zoom]` / `[data-chart-close]`, installed once in `startApp`,
  survives every swap.
- **D5 — Dedicated overlay `#chart-full`, not the existing `#modal`.**
  `.modal-card` is a centered 420px card — wrong shape. The overlay is
  `position:fixed; inset:0` with `background: var(--bg)` (opaque, escapes the
  480px app column — a free desktop bonus), a slim header (Polish title + ✕,
  48px target) and a chart body filling the rest.
- **D6 — Orientation rule: decide from the viewport, not the sensor.**
  On open and on every `resize` (debounced ~150 ms), measure the chart body box:
  - **landscape box** (user rotated, auto-rotate on): render normally, filling it;
  - **portrait box** (auto-rotate off, or not yet rotated): render for the
    *transposed* box and rotate the wrapper 90° with CSS.
  This covers all four combinations of {auto-rotate on/off} × {holding
  portrait/landscape} with zero permissions. `screen.orientation.lock('landscape')`
  was considered and **rejected**: needs the Fullscreen API first on Android and is
  unsupported in iOS Safari — the CSS rotation makes it unnecessary.
- **D7 — Android back button closes the overlay, not the screen.** On open, push a
  dummy history entry (`history.pushState({chartFull:true}, '')` — same URL, so no
  `hashchange`, the router is untouched); a `popstate` listener closes the overlay;
  closing via ✕/Escape calls `history.back()` to consume the entry. **Fallback:**
  if this proves flaky on iOS standalone or under the `/FIRE/` subpath during QA,
  delete the pushState/popstate block — ✕, Escape and `route()` teardown are
  independent of it.
- **D8 — Every data chart gets the button; the dashboard ring does not.** The
  progress ring (`ringSVG`) is a gauge, not a time series. Empty charts (builder
  returned `''`) render nothing, hence no button — `zoomable` guards on that.
- **D9 — No new colors, no manifest change.** The overlay reuses `--bg`/`--line`/
  `--muted`; dark/light follow automatically. `manifest.webmanifest` keeps **no**
  `orientation` field (locking to portrait would kill the rotate-to-landscape flow
  for auto-rotate users).

## Step 1 — new pure module `js/charts.js`

Move `chartSVG`, `stackedBarSVG`, `formatShort` out of `ui.js` verbatim, add a
local `esc()` (same 5-char map as `analysis.js`), then extend the options —
**default output must stay byte-identical** (guarded by F27):

```js
export function chartSVG(rows, defs, { height = 170, width = 440, maxPoints = 120, detail = false } = {})
export function stackedBarSVG(rows, segments, { height = 170, width = 440, detail = false } = {})
```

- `width` replaces the hardcoded `W = 440`; `maxPoints` replaces the hardcoded
  `120` in the decimation step.
- `detail: false` → exactly today's output (0 / half / max Y labels; first/last
  year on X for lines; `labelStep = ceil(n/8)` for bars).
- `detail: true` →
  - **Y axis**: gridlines + labels also at ¼ and ¾ (5 labels total, same
    `formatShort` formatting);
  - **`chartSVG` X axis**: intermediate year labels — target
    `L = max(2, round(width/110))` labels, pick evenly spaced point indices,
    print `ym.slice(0,4)`, skip a label when it repeats the previously printed
    year; first and last keep today's anchors/anchoring;
  - **`stackedBarSVG` X axis**: `labelStep = ceil(n / max(8, round(width/55)))`
    — more year labels on a wide canvas.

`ui.js` then does `import { chartSVG, stackedBarSVG } from './charts.js';` and
deletes the moved code (its exports of these two names disappear with it — nothing
else imported them).

Layering: `charts.js` imports **nothing** → it joins **L0** in the CLAUDE.md
diagram; `ui.js`'s import list gains `charts`.

## Step 2 — tests (new fixture group **F27**, `tests/test-engine.js`)

Pure string assertions on `charts.js` (shared by Node runner and `tests.html`):

1. **Default parity / purity**: fixed synthetic rows (e.g. 36 months, two defs) →
   result contains `viewBox="0 0 440 170"`, exactly 3 `class="axis"` lines, Y
   labels `0` / `formatShort(max/2)` / `formatShort(max)`, first/last year texts;
   calling twice yields the identical string.
2. **Size options**: `{width: 800, height: 360}` → `viewBox="0 0 800 360"`, no
   `NaN` anywhere in the output (regex).
3. **Decimation**: 500 rows with `maxPoints: 240` → each polyline has ≤ 241
   points and the last row's value is included (last point x ≈ `width − padR`).
4. **`detail` on lines**: 5 Y-axis labels; a 30-year series at `width: 800` gets
   ≥ 4 intermediate X year labels with no consecutive duplicates; `detail: false`
   keeps exactly 2 X labels.
5. **`detail` on bars**: 30 yearly rows — default labels every `ceil(30/8)=4`
   years; `{detail: true, width: 800}` labels every `ceil(30/14)=3` or denser;
   default output unchanged vs pre-change expectations.

## Step 3 — `js/ui.js`: registry, button, overlay

### 3a. Registry + `zoomable`

```js
// ── Wykres na pełnym ekranie ────────────────────────────────────────────
const chartZoom = new Map();   // key → { title, build, legendHTML }

function zoomable(key, title, build, { legendHTML = '' } = {}) {
  const html = build({});                       // inline rendering, default opts
  if (!html) return '';                         // pusty wykres → bez przycisku
  chartZoom.set(key, { title, build, legendHTML });
  return `<div class="chart-zoom">${html}
    <button type="button" class="chart-zoom-btn" data-chart-zoom="${key}"
      aria-label="Pokaż wykres na pełnym ekranie">⛶</button></div>`;
}
```

`route()` gains `chartZoom.clear()` and `closeChartFull({ keepHistory: true })`
next to the existing `closeModal()`.

Convert the 13 call sites: each `chartSVG(rows, defs)` /
`stackedBarSVG(rows, segs)` expression becomes
`zoomable(key, title, o => chartSVG(rows, defs, o), { legendHTML })` with the
rows/defs hoisted to a `const` where they aren't already. For the three
swap-updated charts the `zoomable` call already sits inside the result closure
(`dieWithZero` handler, `overpayResult()`, `loanCalcResult()`), so re-registration
on input is automatic.

### 3b. Overlay open/render/close

```js
let chartFullKey = null, chartFullResizeT = null, chartFullKeyHandler = null;

function openChartFull(key) {
  const reg = chartZoom.get(key);
  if (!reg || chartFullKey) return;
  chartFullKey = key;
  history.pushState({ chartFull: true }, '');          // D7 (usuwalny fallback)
  const el = document.getElementById('chart-full');
  el.innerHTML = `<div class="chart-full-head"><b>${esc(reg.title)}</b>
      <button type="button" class="chart-zoom-btn" data-chart-close aria-label="Zamknij">✕</button>
    </div><div class="chart-full-body"></div>`;
  el.hidden = false;
  renderChartFullBody();
  chartFullKeyHandler = e => { if (e.key === 'Escape') closeChartFull(); };
  document.addEventListener('keydown', chartFullKeyHandler);
  window.addEventListener('resize', onChartFullResize);
}
```

`renderChartFullBody()`:

- measure `body.getBoundingClientRect()` (measuring the flex box directly
  sidesteps the iOS `100vh` quirks — never use `window.innerHeight` here);
- `landscape = box.width >= box.height`; long/short = ordered dimensions;
- viewBox: `W = 800`, `H = clamp(round(800 * short / long), 240, 620)`;
- `svg = reg.build({ width: W, height: H, detail: true, maxPoints: 240 })`;
- landscape → `body.innerHTML = svg + legend`; portrait →
  `body.innerHTML = '<div class="cf-rot" style="width:' + Math.round(box.height) + 'px">' + svg + legend + '</div>'`
  — the wrapper is absolutely centered (`left/top 50% + translate(-50%,-50%)
  rotate(90deg)`), so its layout overflow never shifts the visual center.

`onChartFullResize`: 150 ms debounce → `renderChartFullBody()` (the stored
closure re-renders; state can't change while the overlay covers the UI).

`closeChartFull({ keepHistory = false } = {})`: idempotent teardown (hide, clear
`innerHTML`, remove both listeners, `chartFullKey = null`), then
`if (!keepHistory) history.back()` to consume the pushed entry.

### 3c. One-time wiring in `startApp`

```js
document.addEventListener('click', e => {
  const zoomBtn = e.target.closest('[data-chart-zoom]');
  if (zoomBtn) return openChartFull(zoomBtn.dataset.chartZoom);
  if (e.target.closest('[data-chart-close]')) closeChartFull();
});
window.addEventListener('popstate', () => {
  if (chartFullKey) closeChartFull({ keepHistory: true });
});
```

(A hash navigation fires `popstate` **and** `hashchange`; the overlay can only be
open when no navigation is possible underneath it, and both paths funnel into the
idempotent `closeChartFull`, so the double signal is harmless.)

  overlay is visually fullscreen already inside the PWA window.
